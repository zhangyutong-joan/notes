适用场景：刚加入项目、接手一个服务，或者想搞清楚某个协议、数据模型请求链路

**IDE插件工作流(本地探索最快)步骤：**
1. 打开最相关的文件
2. 选中你关心的代码(可选，但推荐)
3. 给 Codex 下 prompt：
中文：
```markdown
解释请求是怎么流经这段选中代码的。

包含:
- 涉及的每个模块，职责的简短总结。
- 哪些数据在哪里被校验。
- 改这段代码时要留神的一两个「坑」

验证方式：
让它给一份你能核对的图或清单：
把请求链路总结成一个带编号的步骤列表，然后列出涉及的文件。
```
英文：
```markdown
Explain how the request flows through the selected code.

Include:
- a short summary of the responsibilities of each module involved
- what data is validated and where
- one or two "gotchas" to watch for when changing this

Verification: Ask for a diagram or checklist you can verify:
Summarize the request flow as a numbered list of steps. Then list the filesinvolved.
```

**CLI工作流(想要完整记录和shell命令时好用):**
1. 启动一个交互会话
2. 附上文件(可选)，然后下prompt:

中文：
我需要理解这个服务用的协议。读一下 @foo.ts和 @schema.ts，解释schema和请求/响应链路。重点讲必填字段和可选字段的区别，以及向后兼容的规则。上下文说明：输入框里可以用`@` 插入工作区的文件路径，或者用`/mention`附上指定文件。

英文：
I need to understand the protocol used by this service. Read @foo.ts@schema.ts and explain the schema and request/response flow. Focus onrequired vs optional fields and backward compatibility rules.
Context notes: You can use @ in the composer to insert file paths from the
workspace, or /mention to attach a specific file.
