## 常见CLI命令
在powershell里面
- `claude`：直接进入会话
- `claude -c` ：继续上次对话
- `claude update`：更新版本
- `claude "【你的query】"`：带初始提示启动 REPL
## 常用斜杠命令
- `/help`：查看所有命令
- `/clear`：清除对话历史，新开一个会话
- `/compact`：压缩当前对话上下文，释放空间
- `/init`：分析项目并生成 **CLAUDE.md 文件**（首次在新项目中使用），没有强制的格式要求
>  **CLAUDE.md 的作用** <br />
>  没有 `CLAUDE.md` 时，Claude 每次都从零开始理解你的项目，你需要反复告诉它：用哪个包管理器、代码风格是什么、测试怎么跑、哪些文件不要动……有了 `CLAUDE.md`，这些信息只需写一次，Claude 每次都会遵守。
>  具体来说，`CLAUDE.md` 可以帮你做到以下几件事：
> 	- **统一团队行为**：将文件提交到 git，所有团队成员使用 Claude Code 时都遵循相同的规范
> 	- **减少重复沟通**：项目约定、架构规则、禁止事项只写一次，永久生效
> 	- **降低出错概率**：明确告知 Claude 哪些操作有风险，避免它做出错误的决策
> 	- **加速 AI 理解**：帮助 Claude 快速定位关键文件和理解项目结构，减少不必要的文件探索<br />
> **CLAUDE.md的维护建议** <br />
	 **1. 保持精简**
	 **2. 持续更新**
	 **3. 用命令式语言**
- `/resume`：从历史中选择并恢复之前的对话
- `/undo`：撤销上一次文件修改（Claude 的修改效果不对时）
- `/redo`：重做被撤销的修改（撤销后发现还是原来的好）
- `/login`：登录或切换账号
- `/config`：查看和修改配置，调整 Claude Code 行为设置
- `/cost`：查看本次会话的 Token 用量
## 其他
- 用 ` @ ` 可以指定文件
- **支持多模态输入**，可以将截图直接粘贴到对话中
- **善用"继续"**：如果 Claude 的回答被截断，或者你觉得它还没说完，直接说"继续"即可
